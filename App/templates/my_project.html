Medical Insurance Charges Prediction

This project is designed to predict medical insurance charges for individuals based on their demographic and lifestyle characteristics, such as age, BMI, smoking status, number of children, and region. The goal is to create a robust, interpretable, and accurate machine learning model that can provide insights to both individuals and insurance companies.

1. Problem Statement

Health insurance premiums often vary significantly due to multiple personal and lifestyle factors. Predicting these costs accurately helps individuals plan finances and enables insurers to estimate fair premiums. The challenge lies in building a predictive model that is not only accurate but also interpretable, so that end-users understand the reasoning behind the predictions.

2. Dataset

The dataset used in this project contains historical records of insurance customers, including:

  • Age – Numerical, representing the person’s age in years.
  • Sex – Categorical, indicating male or female.
  • BMI – Numerical, representing the Body Mass Index.
  • Children – Number of children/dependents covered by the insurance.
  • Smoker – Categorical, indicating if the person is a smoker.
  • Region – Categorical, indicating the region where the person lives.
  • Charges – Numerical target variable representing the annual medical insurance cost.

3. Exploratory Data Analysis (EDA)

We started with exploratory data analysis to understand patterns, detect anomalies, and visualize relationships between features. This included:

  • Distribution plots for numerical variables (age, BMI, charges).
  • Box plots to identify outliers, especially in charges and BMI.
  • Bar charts for categorical features like smoker status and region.
  • Correlation heatmaps to identify strong predictors of charges.

4. Data Cleaning

The dataset was generally clean with no missing values. However, outlier treatment was performed for extreme BMI values and extremely high charges to reduce the impact of anomalies on the model’s performance.

5. Preprocessing Pipeline

To ensure a reproducible and scalable workflow, we implemented a preprocessing pipeline using scikit-learn’s Pipeline and ColumnTransformer. Steps included:

  • Encoding categorical variables using One-Hot Encoding.
  • Scaling numerical variables using StandardScaler.
  • Combining transformations into a single pipeline object for training and testing.

6. Model Selection & Training

Four regression models were considered:

  • Linear Regression
  • Random Forest Regressor
  • XGBoost Regressor
  • Neural Network Model

Hyperparameter tuning was performed using GridSearchCV to optimize performance.

7. Evaluation Metrics

Models were evaluated using:

  • Mean Absolute Error (MAE)
  • Mean Squared Error (MSE)
  • Root Mean Squared Error (RMSE)
  • R² Score

The XGBoost model achieved the best results, with high R² and low RMSE.

8. Model Explainability with SHAP

SHAP (SHapley Additive exPlanations) was used to interpret the model. SHAP summary plots and dependence plots revealed that:

  • Smoking status has the highest impact on charges.
  • BMI and age also significantly affect predictions.
  • Region plays a smaller role in determining cost.

9. Visualization of Results

Residual plots confirmed that the model's errors were randomly distributed, and actual vs predicted plots showed high alignment.

10. Deployment

The final pipeline (preprocessing + XGBoost model) was serialized using joblib and deployed with a Flask web interface. The UI allows users to input their details and receive cost predictions instantly, along with SHAP-based explanations.

11. Conclusion

This project successfully demonstrates how machine learning can be used to predict medical insurance costs accurately while maintaining transparency through explainability techniques.
